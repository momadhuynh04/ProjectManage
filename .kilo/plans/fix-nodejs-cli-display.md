# Fix Display Issues for Node.js CLI Agents (Gemini, Codex, Claude Code)

## Approach: Phase 1 + Phase 2 in parallel

## Phase 1: Environment Variables + PTY Tuning (Quick Fix)

### Changes to backend/main.py:

1. **Environment variables for Node.js agents** (gemini, claude, codex):
   - TERM=xterm-256color
   - COLORTERM=truecolor
   - TERM_PROGRAM=vscode
   - FORCE_COLOR=1
   - NODE_NO_WARNINGS=1

2. **Use ConPTY (backend=0) for ALL agents** - remove the WinPTY fallback

3. **Delayed resize** - send resize 300ms after spawn to fix TUI size detection race

4. **Strip alternate screen buffer** sequences in pty_reader for Node.js CLIs:
   - Remove \x1b[?1049h (enter alt screen)
   - Remove \x1b[?1049l (exit alt screen)
   - Replace with appropriate cursor resets

### Code changes in main.py:

A. In websocket_terminal function, before PTY spawn:
   - Create env dict with terminal vars for Node.js agents
   - Pass env to PTY spawn (pywinpty supports env parameter)

B. Change pty_backend logic:
   - All agents use backend=0 (ConPTY)

C. Add delayed resize after spawn:
   - asyncio.create_task to set_size after 0.3s delay

D. In pty_reader function:
   - Add regex to strip/replace alternate screen buffer sequences for Node.js agents

## Phase 2: node-pty Bridge (Robust Fix)

### New file: backend/pty_bridge.js
- Node.js script that uses node-pty to spawn CLI agents
- Communicates with Python backend via WebSocket
- Handles input/output/resize/kill commands
- Properly handles all Node.js TUI rendering

### Changes to backend/main.py:
- Add NodePTYBridge class that:
  - Spawns pty_bridge.js as subprocess
  - Connects to bridge via WebSocket
  - Forwards data between frontend WebSocket and bridge WebSocket
- In websocket_terminal:
  - For Node.js agents (gemini, claude, codex): use NodePTYBridge
  - For Python/Go agents (kilo, opencode, blackbox): keep pywinpty

### Dependencies:
- npm install node-pty ws (in backend directory)

## Implementation Steps

1. Install node-pty and ws: cd backend, npm init -y, npm install node-pty ws
2. Create backend/pty_bridge.js
3. Modify backend/main.py:
   a. Add env vars injection
   b. Switch to ConPTY for all
   c. Add delayed resize
   d. Add ANSI stripping in pty_reader
   e. Add NodePTYBridge class
   f. Route Node.js agents through bridge
4. Test all 6 CLIs
